CI GitHub Actions — .github/workflows/ci.yml

Déclenché sur push et pull_request vers main / jihed (tout changement de code ou Makefile inclus).

  • quality — image Docker ci : Ruff, Black, Bandit, pip-audit
  • unit — pytest tests/unit
  • integration — docker compose up, attente des /health, puis pytest test_api.py

Localement : make ci reproduit qualité + tests unitaires (sans démarrer la stack).

ML Pipeline — .github/workflows/ml-pipeline.yml

Déclenché par :

  • workflow_dispatch (manuel)
  • schedule cron hebdomadaire (lundi 06:00 UTC)
  • push sur fichiers listés dans paths: (scripts d’entraînement, fixtures CI, workflow, dépendances, Makefile, …)

Étapes : entraînement via scripts/train_models.py dans le conteneur ci, upload artefacts ml/metadata.json et ml/*.pkl.
